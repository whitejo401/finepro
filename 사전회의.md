https://github.com/financedata/financedatareader

퀀트투자
사용자설정 Gem
제공해주신 링크는 한국의 주식 가격, 지수, 환율, 암호화폐 가격 등 금융 데이터를 매우 손쉽게 수집할 수 있게 해주는 파이썬 라이브러리인 FinanceDataReader의 공식 GitHub 저장소입니다.

이 라이브러리는 공공 데이터 포털이나 개별 거래소의 복잡한 API 사용법을 몰라도, 코드 몇 줄만으로 방대한 금융 데이터를 Pandas DataFrame 형식으로 가져올 수 있어 국내 데이터 분석가와 투자자들 사이에서 매우 인기가 높습니다.

주요 특징과 사용법을 요약해 드립니다.

1. 주요 특징
국내외 주가 데이터: KOSPI, KOSDAQ, KONEX뿐만 아니라 NYSE, NASDAQ, AMEX, S&P 500 등 미증시 데이터도 지원합니다.

상장 종목 리스트: 현재 시장에 상장된 종목들의 리스트와 종목 코드를 쉽게 조회할 수 있습니다.

기타 금융 데이터: 원달러 환율, 금값, 암호화폐(업비트 기준) 가격, 국채 수익률 등 다양한 경제 지표를 제공합니다.

Pandas 호환: 결과값이 Pandas DataFrame으로 반환되어 즉시 분석이나 시각화에 활용 가능합니다.

2. 설치 방법
터미널(또는 CMD)에서 아래 명령어로 설치할 수 있습니다.

Bash
pip install finance-datareader
3. 기본 사용 예시
[종목 리스트 가져오기]
한국거래소(KRX)에 상장된 전체 종목 리스트를 불러옵니다.

Python
import FinanceDataReader as fdr

# 한국거래소 상장 종목 전체
df_krx = fdr.StockListing('KRX')
print(df_krx.head())
[주가 데이터 가져오기]
특정 종목(예: 삼성전자 '005930')의 특정 기간 주가를 가져옵니다.

Python
# 삼성전자(005930) 주가, 2023년부터 현재까지
df = fdr.DataReader('005930', '2023-01-01')
df['Close'].plot() # 종가 시각화
[미국 주식 및 지수]

Python
# 애플(AAPL), 2023년 데이터
apple = fdr.DataReader('AAPL', '2023')

# S&P 500 지수 (US500), 나스닥 (IXIC), 코스피 (KS11)
kospi = fdr.DataReader('KS11', '2023')

실시간 데이터도 있나

퀀트투자
사용자설정 Gem
FinanceDataReader는 기본적으로 '일봉(Daily)' 데이터 수집에 최적화된 라이브러리입니다. 결론부터 말씀드리면, 초 단위나 호가 창을 보여주는 완전한 실시간 데이터는 지원하지 않습니다.

하지만 상황에 따라 '실시간'에 가깝게 활용할 수 있는 부분들이 있으니 아래 내용을 참고해 보세요.

1. 데이터 업데이트 주기
국내 주식: 장 중에는 네이버 파이낸스 등의 데이터를 가져오기 때문에 약 20분 지연된 시세 혹은 당일의 실시간에 가까운 데이터를 반영하기도 하지만, 기본적으로 제공하는 최소 단위는 **1일(Daily)**입니다.

미국 주식: 야후 파이낸스 등을 통해 데이터를 가져오며, 이 역시 장 중 시세를 반영하긴 하나 1분/5분 단위의 분봉 데이터는 제공하지 않습니다.

암호화폐: 빗썸이나 업비트 API를 활용하므로 비교적 최신 가격을 가져오지만, 라이브러리 구조상 '스트리밍' 방식이 아니라 코드를 실행하는 시점의 데이터를 가져오는 방식입니다.

2. 실시간/분봉 데이터가 필요하다면?
만약 단타 매매 전략을 세우거나 초 단위의 데이터가 필요하다면 FinanceDataReader 대신 아래와 같은 대안을 사용해야 합니다.

국내 주식 (완전 실시간/분봉):

증권사 API: 키움증권(Open API+), 대신증권(Cybos Plus), 한국투자증권(REST API) 등을 사용해야 합니다. (보통 윈도우 환경 권장)

pykrx: FinanceDataReader와 유사하지만 국내 주식에 특화되어 있으며, 특정 시점의 시세를 빠르게 조회할 때 유용합니다.

미국 주식 (실시간/분봉):

yfinance: interval 옵션을 통해 1분, 5분, 15분 단위의 데이터를 가져올 수 있습니다.

Alpaca / Polygon.io: 전문적인 실시간 데이터 API 서비스입니다.

3. FinanceDataReader의 용도 (추천)
FinanceDataReader는 "실시간 트레이딩"보다는 다음과 같은 **"퀀트 분석"**에 더 적합합니다.

과거 데이터 백테스팅: 수년치 일별 주가 데이터를 받아 전략 검증

상관관계 분석: 환율, 지수, 개별 종목 간의 흐름 비교

리밸런싱: 주 단위 혹은 월 단위로 포트폴리오 비중을 조절할 때

한 줄 요약: > "장 마감 후 분석이나 일 단위 스윙 투자를 위한 데이터 수집에는 최고지만, 초 단위 실시간 단타용으로는 부적합합니다."

혹시 실시간 데이터가 꼭 필요하신 이유가 무엇인가요? (예: 자동매매 봇 제작, 장중 알림 등) 목적을 알려주시면 그에 맞는 라이브러리를 추천해 드릴 수 있습니다.

FinanceDataReader류의 다른 오픈소스나 증시데이터를 얻어오는 더 좋은 방법이 있나

퀀트투자
사용자설정 Gem
FinanceDataReader(FDR)는 국내외 데이터를 통합해서 보기 좋지만, 특정 목적(국내 특화, 글로벌 전문성, 대량 데이터 등)에 따라 더 강력한 대안들이 존재합니다.

2026년 현재 퀀트 투자자들이 가장 많이 사용하는 목적별 대체 도구를 정리해 드립니다.

1. 국내 주식 데이터의 끝판왕: pykrx
국내 주식(KOSPI, KOSDAQ)에만 집중한다면 FDR보다 훨씬 강력합니다.

특장점: 한국거래소(KRX) 웹사이트를 직접 스크래핑하여, FDR이 주지 못하는 세세한 정보를 제공합니다.

제공 데이터: 종목별 PER, PBR, 배당수익률(가장 강력한 기능), 투자자별 순매수 현황(기관/외국인), 공매도 잔고 등.

추천 상황: 국내 주식 퀀트 팩터(가치 지표) 분석을 할 때 필수입니다.

설치: pip install pykrx

2. 글로벌 표준 및 분봉 데이터: yfinance
전 세계 금융 데이터의 표준인 Yahoo Finance를 기반으로 합니다.

특장점: 미국 등 해외 주식 데이터가 매우 정확하며, FDR과 달리 1분/5분 단위의 분봉(Intraday) 데이터를 무료로 가져올 수 있습니다. (최근 60일 이내 데이터 한정)

제공 데이터: 재무제표, 애널리스트 추천가, 기업 실적 발표 일정 등.

추천 상황: 미국 주식 분석이나 장중 변동성(분봉) 데이터가 필요할 때.

설치: pip install yfinance

3. 차세대 오픈소스 금융 터미널: OpenBB SDK
단순 라이브러리를 넘어 블룸버그 터미널의 오픈소스 버전을 지향하는 거대 프로젝트입니다.

특장점: 주식뿐만 아니라 암호화폐, 경제 지표(FRED), 옵션, 대체 데이터(다크풀 등)를 하나의 SDK로 통합했습니다. 2026년 현재 AI 에이전트 연동 기능이 강화되어 가장 트렌디한 도구입니다.

추천 상황: 전문적인 퀀트 분석 환경을 구축하거나 다양한 데이터 소스를 하나로 묶고 싶을 때.

설치: pip install openbb

4. 가장 "정확하고 빠른" 방법: 증권사 공식 API (REST API)
스크래핑 방식인 위 라이브러리들과 달리, 증권사가 서버 문을 열어주는 공식 통로입니다.

대상: 한국투자증권 API, 키움증권 Open API+ 등.

장점: * 실시간성: 지연 없는 틱(Tick) 시세와 체결 데이터.

신뢰도: 스크래핑 방식은 웹사이트 구조가 바뀌면 깨지지만, API는 안정적입니다.

매매 가능: 데이터 수집뿐만 아니라 실제 주문을 넣을 수 있습니다.

단점: 계좌가 있어야 하고 설정이 다소 번거롭습니다.

증권사API를 통하여 얻어온 정보도 상업적으로 사용가능한가

퀀트투자
사용자설정 Gem
증권사 API를 통해 얻은 데이터를 상업적으로 활용하는 것은 **"매우 까다롭고, 원칙적으로는 금지되어 있다"**고 보시는 것이 안전합니다.

라이브러리(FDR, pykrx 등)의 라이선스와는 별개로, 데이터의 '소유권'은 거래소(KRX)와 증권사에 있기 때문입니다. 상세 내용을 정리해 드립니다.

1. 상업적 이용이 금지되는 대표적인 행위
증권사가 무료 또는 개인 고객용으로 제공하는 API(Open API 등)를 사용해 다음과 같은 행위를 하면 약관 위반 및 저작권 침해에 해당할 수 있습니다.

데이터 재판매: 수집한 시세 데이터를 DB화하여 타인에게 유료로 판매하는 행위.

유료 서비스 노출: API로 받은 실시간 시세를 본인의 유료 앱이나 웹사이트에 그대로 띄워 서비스하는 행위.

가공 데이터 판매: 시세를 가공하여 만든 지표(예: 자체적인 종목 추천 점수 등)를 판매할 때, 그 바탕이 되는 시세 데이터가 무단 사용된 경우 문제가 될 수 있습니다.

2. 왜 제한되나요? (데이터의 원소유주)
증권사도 한국거래소(KRX)나 해외 거래소(나스닥 등)에 **'정보 이용료'**를 지불하고 데이터를 받아와 고객에게 서비스하는 것입니다.

개인용 API: 증권사가 고객 서비스 차원에서 이용료를 대신 내주는 것이므로 '개인적 투자 목적'으로만 한정합니다.

상업용 API: 만약 데이터를 이용해 사업을 하려면, 증권사가 아닌 한국거래소(KRX)와 직접 데이터 수신 계약을 맺고 매월 수백만 원 이상의 정보 이용료를 지불해야 합니다.

3. 허용되는 범위 (Safe Zone)
개인적 용도: 본인의 자동매매 프로그램을 만들거나, 개인적인 퀀트 전략을 검증하는 행위는 100% 허용됩니다.

비영리 연구/교육: 데이터를 직접 배포하지 않고 분석 결과물(차트 이미지, 통계 수치 등)만 공유하는 것은 통상적으로 용인됩니다.

콘텐츠 제작: 유튜브나 블로그에서 분석 결과를 보여주는 것은 괜찮지만, 실시간 시세 중계 등은 저작권 신고 대상이 될 수 있습니다.

4. 실물 경기 선행 지표 (Leading Indicators)

구리 가격 (Dr. Copper): 구리는 모든 산업 전반에 쓰이기 때문에, 구리 가격이 오르면 실물 경기가 살아나고 있다는 강력한 증거가 됩니다.

5. 정치적 이벤트 및 정책 (Policy)

1단계: 시장을 4등분하는 '핵심 축' 세우기 (메릴린치 투자 시계)

성장 지표 (경기): GDP, PMI(구매관리자지수), 실업률, 수출입 동향 등

물가 지표 (인플레이션): CPI, 유가, 원자재 가격, 환율 등

이 두 가지가 오르고 내림에 따라 시장은 4가지 국면으로 나뉩니다. 내가 방금 본 뉴스가 이 4가지 국면 중 어디로 시장을 밀어붙이는지 파악하는 것이 첫 번째입니다.

2단계: 종속 변수(타겟) 확정하기

나쁜 타겟: "내일 장이 좋을까?"

좋은 타겟: "KOSPI 200 지수의 향후 1개월 수익률", "나스닥 기술주 대비 다우 가치주의 상대 수익률"

타겟이 명확해야 수집한 뉴스나 지표(X)가 타겟(Y)에 어떤 영향을 미쳤는지 통계적으로 검증(백테스팅)할 수 있습니다.

금리 (Interest Rates):

미 연준 및 한국은행 기준금리: 모든 자산 가격의 기본 중력 (비정기/월간)

미 국채 수익률 (2년물, 10년물, 30년물): 장단기 금리차(10년물 - 2년물)를 계산하여 경기 침체를 예측하는 데 필수 (일간)

유동성 및 신용 (Liquidity & Credit):

하이일드 스프레드 (High Yield Spread): 우량채와 투기등급 채권의 금리 차이. 시장의 '부도 공포'를 측정 (일간)

M2 통화량: 시중에 풀린 현금의 양 (월간)

3. 주요 자산 가격 및 시장 지표 (Market Pricing)
매크로 변수에 반응하여 실제로 돈이 오가는 시장의 가격 데이터입니다. (일간 실시간)

주식 지수 (Equities): S&P 500, NASDAQ, 다우존스, KOSPI (대형주 흐름), 러셀 2000 (미국 중소형주 - 내수 경기 반영)

외환 (FX):

달러 인덱스 (DXY): 글로벌 안전 자산 선호도의 척도

원/달러 환율, 엔/달러 환율: 수출입 기업의 이익과 외국인 수급에 직결

원자재 (Commodities):

WTI 원유 / 브렌트유: 물가(CPI)에 가장 직접적인 타격을 주는 지표

금 (Gold): 인플레이션 헤지 및 지정학적 위기 시 방어 자산

구리 (Dr. Copper): 실물 경기의 바로미터

대체 자산: 비트코인 (글로벌 잉여 유동성의 가장 민감한 척도로 활용됨)

4. 시장 심리 및 포지셔닝 (Sentiment & Positioning)
인간의 탐욕과 공포, 그리고 기관 투자자들의 실제 베팅 방향을 보여주는 데이터입니다.

VIX (변동성 지수): 주식 시장의 공포 지수 (일간)

MOVE 지수: 채권 시장의 공포/변동성 지수 (일간)

Put/Call Ratio: 옵션 시장에서 하락(Put)에 베팅하는 비율 (일간)

COT 리포트 (Commitments of Traders): 선물 시장에서 헤지펀드 등 투기 세력의 매수/매도 포지션 동향 (주간)

5. 글로벌 공급망 및 지정학적 대체 데이터 (Alternative Data)
최근 글로벌 퀀트 펀드들이 가장 중요하게 보는 '뉴스 외적인' 수치들입니다.

SCFI (상하이컨테이너운임지수) / BDI (발틱운임지수): 글로벌 물류 비용과 공급망 병목 현상 파악 (주간)

EPU (경제정책 불확실성 지수): 뉴스 기사 등을 자연어 처리(NLP)하여 현재 정책의 불확실성을 수치화한 데이터 (월간/일간)

💡 퀀트 투자자의 다음 스텝을 위한 조언
위 리스트를 보시면 일간(Daily) 데이터부터 분기별(Quarterly) 데이터까지 **'시간의 주기'**가 다릅니다. 이 서로 다른 주기의 데이터를 하나의 표(DataFrame)로 합치는 과정(주로 빈 칸을 이전 값으로 채우는 Forward Fill 방식 사용)이 퀀트 모델링의 첫 번째 기술적 장벽입니다.

이 방대한 데이터들을 수집하기 위해, 파이썬 API(FRED, yfinance 등)를 이용한 '자동 수집 파이프라인(코드)'부터 바로 구축해 보시겠습니까, 아니면 이 중 특정 지표들의 과거 데이터를 먼저 엑셀/CSV로 다운받아 눈으로 흐름과 상관관계를 확인해 보는 것이 좋으신가요?
