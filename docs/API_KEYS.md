# API 키 발급 및 사용 조건 가이드

발급 후 `.env` 파일에 입력한다. `.env.example`을 복사해서 사용.

> **면책 고지**: 이 문서는 각 서비스의 공개 이용약관을 기반으로 작성됐으나, 약관은 수시로 변경된다.
> 상업적 사용 전에는 반드시 각 서비스의 최신 이용약관을 직접 확인할 것.

## 블로그 게시 및 광고수익 가능 여부

> **전제**: 광고(애드센스 등)가 붙은 블로그는 수익 창출 목적으로, 대부분의 API 약관에서 **상업적 사용**으로 분류된다.

### 핵심 구분 기준

| 콘텐츠 유형 | 설명 | 일반적 허용 여부 |
|-----------|------|---------------|
| **분석 결과** | 수집 데이터를 가공한 차트, 인사이트, 해석 | 대부분 허용 |
| **원시 데이터 그대로 게시** | API 응답값을 테이블/CSV로 그대로 올리는 행위 | 제한적 |
| **뉴스/게시물 원문 인용** | 뉴스 기사, Reddit 게시물 원문 복사 | 저작권 문제 |

### API별 블로그 광고수익 가능 여부

| API | 광고수익 블로그 | 조건 | 비고 |
|-----|-------------|------|------|
| **DART** | ✅ 가능 | 출처 표기 필수 | 공공누리 제1유형 |
| **ECOS** | ✅ 가능 | 출처 표기 필수 | 공공누리 제1유형 |
| **공공데이터포털** | ✅ 가능 (제1유형) | 출처 표기 필수 | 제2유형 데이터는 불가 |
| **국토교통부** | ✅ 가능 | 출처 표기 필수 | 공공누리 제1유형 |
| **FRED** | ✅ 가능 | 출처 표기 권장 | 분석 결과 기준 |

| **KIS** | ❌ 불가 | - | 본인 계좌 전용, 제3자 콘텐츠 금지 |

| **Reddit 무료** | ❌ 불가 | 유료 계약 필요 | 2023년 정책 변경 |

### pykrx / KRX 데이터 별도 주의

pykrx는 KRX(한국거래소) 웹사이트에서 데이터를 수집한다. KRX 데이터의 저작권은 한국거래소에 귀속된다.

| 사용 형태 | 허용 여부 |
|---------|---------|
| 개인 투자 분석, 내부 사용 | ✅ 허용 |
| 비상업적 블로그 (광고 없음) | ✅ 허용 |
| 광고수익 블로그에 KOSPI 차트 게시 | ⚠️ 그레이존 |
| KRX 데이터 기반 유료 서비스 | ❌ KRX와 별도 계약 필요 |

실무적으로 수많은 개인 블로그가 KOSPI/종목 차트를 게시하지만, 엄밀히는 KRX 약관 위반 소지가 있다. 상업적 서비스 구축 시에는 KRX 데이터 이용 계약(https://data.krx.co.kr)을 체결하거나 yfinance(^KS11, ^KQ11)로 대체하는 것을 권장한다.

### 분석 결과 vs 원시 데이터 — 법적 해석

```
원시 데이터를 그대로 게시       → 저작권/약관 위반 가능성 높음
             ↓ 가공
수집 → 분석 → 차트/인사이트    → 대부분 허용 (2차 저작물)
```

분석 결과(차트, 통계, 해석)는 **사실 데이터를 가공한 2차 창작물**로 보호받으며, 원본 데이터의 저작권과 별개로 취급되는 것이 일반적이다. 단, 수치 자체(예: "삼성전자 오늘 종가 62,000원")는 사실(fact)로 저작권이 없으나, 데이터베이스 형태로 대량 복제·배포하는 것은 데이터베이스 권리(sui generis right)로 보호될 수 있다.

### 안전한 운영 가이드

1. **원시 데이터 대량 게시 금지**: API 응답을 테이블 형태로 그대로 올리지 않는다
2. **출처 표기 철저**: 공공데이터는 반드시 출처 명시 (예: "출처: 한국은행 ECOS")
3. **상업화 전 유료 전환**: CoinGecko, Glassnode, NewsAPI는 광고수익 블로그 운영 전 유료 플랜으로 전환
4. **면책 문구 추가**: "본 블로그의 데이터는 투자 조언이 아님" 명시
5. **약관 주기적 확인**: 특히 Reddit(2023년 변경 사례처럼) API 정책은 수시로 변경됨

## 1. DART (전자공시)

**용도**: 재무제표, 배당공시, 사업보고서 (dart-fss)  
**무료**: 무제한 | **발급 소요**: 즉시

### 사용 조건
- **라이선스**: 공공누리 제1유형 (출처표시)
- **개인 사용**: 제한 없음
- **상업적 사용**: 허용. 서비스/제품에 활용 가능
- **데이터 재배포**: 원출처(금융감독원 DART) 표기 시 가능
- **금지 사항**: DART 데이터를 유사 공시 서비스로 그대로 재판매하는 행위
- **약관 확인**: https://opendart.fss.or.kr/intro/about.do

### 발급 방법
1. https://opendart.fss.or.kr 접속
2. 우측 상단 **인증키 신청/관리** 클릭
3. 회원가입 (이메일 인증)
4. 로그인 후 **인증키 신청** → 용도 입력 후 즉시 발급

```env
DART_API_KEY=your_40char_key_here
```

> 일 10,000건 제한. 재무제표 전체 수집 시 여러 날로 나눠 실행 권장.

## 2. 한국투자증권 KIS

**용도**: 실시간 시세, 분봉 데이터  
**무료**: 계좌 개설 필수 | **발급 소요**: 1~2 영업일

### 사용 조건
- **개인 사용**: 허용 (본인 계좌 기반 투자 및 분석)
- **상업적 사용**: **원칙적 금지**. 제3자에게 데이터를 제공하거나 서비스화하는 것은 별도 계약 필요
- **데이터 재배포/재판매**: **금지**
- **허용 범위**: 본인 계좌 투자 목적의 시스템 매매, 개인 리서치 도구
- **주의**: 불특정 다수에게 시세 데이터를 제공하는 서비스 구축 시 한국거래소(KRX) 데이터 이용 계약 별도 체결 필요
- **약관 확인**: https://apiportal.koreainvestment.com (서비스 이용약관)

### 발급 방법
1. https://apiportal.koreainvestment.com 접속
2. 한국투자증권 계좌 개설 (비대면 가능)
3. 사이트 로그인 후 **앱 등록** → **모의투자** 선택
4. APP KEY / APP SECRET 발급

```env
KIS_APP_KEY=your_app_key
KIS_APP_SECRET=your_app_secret
```

> 현재 프로젝트는 EOD 데이터만 사용하므로 KIS는 선택사항. pykrx로 대체 가능.

## 3. ECOS (한국은행)

**용도**: 기준금리, M2, GDP, CPI (한국)  
**무료**: 무제한 | **발급 소요**: 즉시

### 사용 조건
- **라이선스**: 공공누리 제1유형 (출처표시)
- **개인 사용**: 제한 없음
- **상업적 사용**: 허용. 출처(한국은행 경제통계시스템) 표기 조건으로 상업적 활용 가능
- **데이터 재배포**: 원출처 표기 시 가능
- **금지 사항**: 한국은행의 공식 입장인 것처럼 오해를 유발하는 방식으로 사용하는 행위
- **약관 확인**: https://ecos.bok.or.kr (이용 가이드 > 이용 약관)

### 발급 방법
1. https://ecos.bok.or.kr 접속
2. 회원가입 → 로그인
3. **OPEN API** → **인증키 신청** → 즉시 발급

```env
ECOS_API_KEY=your_key_here
```

> 일 10,000건 제한. 주요 통계 코드: 722Y001(기준금리), 901Y009(CPI), 200Y001(GDP).

## 4. 공공데이터포털

**용도**: 상장법인 목록, 기업공시 정보  
**무료**: 무제한 | **발급 소요**: 즉시~1 영업일

### 사용 조건
- **라이선스**: 데이터별로 공공누리 유형이 다름 (각 API 상세 페이지에서 확인)
  - 제1유형 (출처표시): 상업적 사용 허용 — 대부분 해당
  - 제2유형 (출처표시+상업금지): 상업적 사용 **불가**
  - 제4유형 (출처표시+변경금지): 원본 그대로만 사용 가능
- **개인 사용**: 제한 없음
- **상업적 사용**: 제1유형 데이터는 허용. 활용 전 각 API의 라이선스 유형 반드시 확인
- **데이터 재배포**: 유형에 따라 다름. 원출처 표기 필수
- **약관 확인**: https://data.go.kr (이용약관, 각 API 상세 페이지 라이선스 표기)

### 발급 방법
1. https://data.go.kr 접속 → 회원가입 → 로그인
2. 원하는 API 검색 → **활용신청**
3. **마이페이지** → **오픈API** → **인증키** 확인

```env
DATA_GO_KR_API_KEY=your_key_here
```

> 하나의 키로 공공데이터포털 내 모든 API 사용 가능.

## 6. FRED (미국 연준)

**용도**: 미국 CPI, GDP, 금리, 실업률, M2 등 거시지표  
**무료**: 무제한 | **발급 소요**: 즉시

### 사용 조건
- **개인 사용**: 제한 없음
- **상업적 사용**: 허용. 내부 분석, 리포트, 서비스에 활용 가능
- **데이터 재배포**:
  - 소량 인용/시각화: 출처(Federal Reserve Bank of St. Louis) 표기 시 허용
  - 대량 재배포 또는 경쟁 데이터 서비스 구축: **별도 라이선스 계약 필요**
- **출처 표기**: 권장 형식 — `Source: Federal Reserve Bank of St. Louis (FRED)`
- **주의**: FRED 데이터 자체는 공공 도메인이나, 일부 시리즈는 제3자 제공 데이터로 별도 약관 적용 (각 시리즈 페이지에서 "Source" 확인)
- **약관 확인**: https://fred.stlouisfed.org/legal

### 발급 방법
1. https://fred.stlouisfed.org → **My Account** → **Create an Account**
2. 로그인 후 **My Account** → **API Keys** → **Request API Key**
3. 용도 입력 → 즉시 발급

```env
FRED_API_KEY=your_32char_lowercase_key
```

> 키는 반드시 32자 소문자 영숫자. 일 120,000 요청 제한 (사실상 무제한).

## 7. EIA (미국 에너지정보청)

**용도**: 원유재고, 원유생산량, 천연가스 가격  
**무료**: 무제한 | **발급 소요**: 수분 내

## 11. Reddit

**용도**: 커뮤니티 감성 분석 (r/wallstreetbets, r/investing 등)  
**무료**: 소규모 비상업적 | **발급 소요**: 즉시

### 사용 조건

> Reddit은 2023년 6월 API 정책을 대폭 변경했다. 상업적 사용 전 반드시 최신 약관 확인 필요.

| 사용 형태 | 허용 여부 |
|---------|---------|
| 개인 비상업적 (일 100,000 요청 이하) | ✅ 무료 |
| 비상업적 연구/학술 목적 | ✅ 무료 |
| 상업적 사용 (수익 창출 서비스) | ❌ 유료 계약 필요 |
| AI 학습 데이터 수집 | ❌ 별도 계약 필요 |
| 데이터 재판매 | ❌ 금지 |
| 접근성 앱 (장애인 지원 등) | ✅ 무료 예외 |

- **상업적 사용 기준**: 서비스를 통해 수익을 창출하거나 기업 내부 비즈니스 목적으로 사용하는 경우
- **대량 수집**: 일 100,000 API 요청 초과 시 유료 Data API 계약 필요
- **콘텐츠 저작권**: Reddit 게시물의 저작권은 원 작성자에게 귀속. 원문 그대로의 재배포 금지
- **상업적 이용 문의**: https://reddit.com/r/reddit/comments/12qwagm
- **약관 확인**: https://www.redditinc.com/policies/data-api-terms

### 발급 방법
1. https://www.reddit.com/prefs/apps 접속
2. **create another app** → type: **script** 선택
3. redirect uri: `http://localhost:8080`
4. **create app** → CLIENT_ID (앱 이름 아래 짧은 문자열), CLIENT_SECRET (secret 항목)

```env
REDDIT_CLIENT_ID=your_14char_client_id
REDDIT_CLIENT_SECRET=your_27char_secret
REDDIT_USER_AGENT=financial-data-bot/1.0 by u/your_username
```

> `REDDIT_USER_AGENT` 형식: `앱이름/버전 by u/레딧유저명` — Reddit 정책상 필수

## 키 입력 방법

`.env`에 주석 없이 값만 입력:

```bash
# 잘못된 입력 (주석이 값으로 인식됨)
FRED_API_KEY=           # https://fred.stlouisfed.org

# 올바른 입력
FRED_API_KEY=abc123def456...
```

입력 후 동작 확인:
```bash
python -c "from config import FRED_API_KEY; print(bool(FRED_API_KEY))"
# True 출력되면 정상
```
